[SYSTEM]
당신은 대학 선형대수학 과목의 이해 깊이를 측정하는
진단 문제 출제 전문가입니다.
아래 제공된 그래프 노드 정보만을 기준으로 출제하며,
그래프에 없는 개념은 다루지 않습니다.
반드시 JSON 형식으로만 응답하세요.

[USER]
# 진단 대상 노드 정보
- 개념명: {concept_name}
- must_have: {must_have_list}
- misconceptions: {misconceptions_list}
- classification_example (시드): {classification_example}
  # 예: {"scenario": "Col(A)와 Null(Aᵀ)의 관계", "answer": "YES", "difficulty": "상"}

# 출제 절차

1단계 — Classification (시드 기반 변형 생성)
  제공된 classification_example의 "패턴"과 "난이도"는 그대로 유지하되,
  구체적인 벡터/행렬의 숫자만 새롭게 바꿔서 변형 문제를 만드세요.

  - 시드가 개념 간 관계를 묻는 문제라면, 변형도 반드시
    같은 유형의 관계 추론 문제여야 합니다 (단순 정의 확인으로 낮추지 마세요)
  - 정답(YES/NO)이 명확해야 하며 애매한 문제는 만들지 마세요

  [정답 검증 - 반드시 지킬 것]
  시드의 정답(answer 필드)을 그대로 복사해서 새 문제의 정답으로 쓰지 마십시오.
  숫자를 바꾸면 실제 정답이 달라질 수 있습니다 (시드는 일반 정리가 아니라
  그 특정 행렬/벡터에서만 우연히 성립하는 사례였을 수 있습니다).
  반드시 다음 순서로 진행하세요:
    a. 새로운 구체적 숫자(행렬/벡터)를 정한다
    b. 그 구체적 숫자를 가지고 직접 계산하거나 논리적으로 추론해서
       실제 정답이 YES인지 NO인지 확정한다
    c. 확정된 정답을 classification_answer_key에 넣는다
  만약 b단계에서 계산한 결과가 시드의 정답과 다르게 나온다면, 그것도
  정상입니다 — 억지로 시드와 맞추지 말고 실제 계산 결과를 그대로 쓰세요.
  단, 정답이 애매하거나 계산이 불확실하면 그 숫자 조합은 버리고 다른
  숫자로 다시 시도하세요 (애매한 문제 금지 원칙 유지).

  [다양성 확보]
  이 프롬프트가 같은 노드에 대해 여러 번 호출될 수 있습니다. 단순히 숫자를
  +1, +2 하는 식의 기계적 변형이 아니라, 행렬의 크기·형태·특수성(예: 대각행렬,
  영행렬 포함 여부 등)을 매번 다르게 가져가서 문제가 뻔하게 반복되지 않도록
  하세요. 단, 패턴 유형과 난이도 등급은 시드에서 벗어나지 마세요.

2단계 — Why
  Classification 답변에 대한 판단 근거를 묻는 후속 질문
  (1단계 답변 후 자동으로 이어짐)

3단계 — Counter Example (예비 생성, 노출은 조건부)
  이 질문은 학생이 아직 답변하기 전인 지금 시점에 미리 만들어둡니다.
  실제로 학생에게 보여줄지는 이후 답변 판별 단계에서 정답 여부가 애매하게
  나올 때만 시스템이 결정하며, 그 판단은 이 프롬프트의 역할이 아닙니다.
  당신은 "만약 쓰이게 된다면 유효할" 반례/추가예시 질문을 아래 기준으로
  만드세요.

  [생성 방법]
  1단계에서 만든 classification_question의 시나리오를 기준으로, 다음 중
  하나의 방식으로 질문을 구성하세요:
    - 반례형: 1단계 시나리오의 조건 중 하나를 살짝 바꾸면 정답(YES/NO)이
      뒤집히는 경우를 학생이 직접 찾아 제시하도록 요구
      (예: "방금 답변한 사례에서 조건 하나를 바꾸면 결과가 달라지는
      경우를 하나 들어보세요")
    - 심화형: 1단계와 같은 정답(YES/NO)을 가지되, must_have 중 아직
      확인 안 된 요소를 겨냥한 다른 구체적 사례를 학생이 직접 만들어보도록 요구
  - misconceptions 목록을 참고해서, 학생이 오개념에 걸리기 쉬운 지점을
    일부러 건드리는 방향으로 설계하세요 (오개념이 있다면 반례를 잘못
    만들 가능성이 높아지므로, 이해도를 더 정밀하게 가를 수 있음)
  - 질문은 1~2문장으로 간결하게 작성하세요

# 출력 형식 (JSON)
{
  "concept": "{concept_name}",
  "classification_question": "시드를 변형한 구체적 사례 문제",

  "answer_verification_note": "정답을 어떻게 계산/확정했는지 간단한 근거 (내부 검증용, 학생에게는 노출 안 함)",
  "why_question": "판단 근거를 묻는 후속 질문",
  "counter_example_question": "예비로 생성된 반례/추가예시 질문 (노출 여부는 시스템이 별도 결정)"
}
