# 판별 기준

## 축 1. 필수 요소(must_have) 충족 — 의미 기반 판단
각 must_have 항목에 대해 다음 절차로 판단하세요.
  1) 학생 답변에서 해당 개념을 가리키는 표현을 찾는다
     (정확히 같은 단어가 아니어도 됨 — 동의어, 풀어쓴 표현 인정)
  2) 그 표현이 must_have가 의미하는 핵심 개념과 일치하는지 확인
     - 단, 다른 개념과 혼동되는 부정확한 표현은 불인정
       (예: "스칼라"를 "크기"라고 한 경우 — norm과 혼동 가능성 있어 불충족)
  3) 애매하면(판단이 50:50이면) 충족하지 않음으로 처리
     (관대한 판정보다 정확한 진단이 우선)

## 축 2. 오개념(misconceptions) 포함 — 유연한 패턴 매칭
misconceptions에 적힌 문장과 토씨가 달라도,
같은 패턴의 혼란이 감지되면 오개념으로 분류하세요.
예: "벡터 직교와 부분공간 직교를 동일시함"이 등록되어 있을 때,
    "직교하면 선형독립"이라는 답변도 같은 유형의 혼동으로 간주

## 축 3. Classification 정답 여부
classification_answer_key와 학생의 답변이 일치하는지 확인

## 판정 규칙 (아래 순서대로 확인, 먼저 해당하는 것으로 확정)

1) Classification 오답 → **실패**
2) Classification 정답 이지만 오개념 발견 → **실패**
   (오개념은 "근거가 약함"과 다른 신호입니다. 정답을 맞혔더라도 잘못된
   믿음을 갖고 있다는 뜻이므로, 근거 부족보다 심각하게 취급합니다.)
3) Classification 정답 + 오개념 없음 + must_have 충족 비율이 1/3 이하 → **실패**
4) Classification 정답 + 오개념 없음 + must_have 충족 비율이 1/3 초과 ~
   전체 미만 → **부분**
   (정답 자체는 맞혔으니 "모른다"가 아니라 "아직 얕게 안다"는 신호입니다.
   이 경우는 선행개념에 원인이 있다기보다 이 노드 자체를 더 다져야 하는
   상태이므로, 선행노드 추적 없이 여기서 확정합니다.)
5) Classification 정답 + 오개념 없음 + must_have 전체 충족 → **통과**

## next_action 매핑 (판정에 따라 고정, 임의 선택 금지)
- 통과 → "통과_종료"
- 부분 → "부분_공백기록"
- 실패 → "분석_필요" (프롬프트 ③으로 전달, 선행노드 탐색 시작)

## gap_type 태깅
- 통과 → null
- 부분 → "shallow" (근거/설명이 얕음, 이 노드 자체를 보강 대상으로)
- 실패 → "conceptual" (개념 자체에 오류 또는 공백, ③에서 확정되는 원인
  노드에 최종적으로 붙는 태그. 이 프롬프트 단계에서는 "conceptual_pending"으로
  표시해 아직 원인 노드가 확정 안 됐음을 나타냅니다.)
